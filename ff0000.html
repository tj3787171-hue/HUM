Peer veth chain guidance

The merger plot expects the developer proxy chain to exist before you start tracing behavior. Use the defaults from scripts/hum-dev-netns.sh unless you have overridden them with HUM_* variables.

root namespace hum-proxy-host0 10.200.0.1/30 fe80::1/64
⇄ peer veth ⇄
proxy namespace hum-proxy-ns0 10.200.0.2/30 fe80::2/64
netns hum-proxy-ns default via 10.200.0.1
→ downstream hint →
dummy endpoint hum-dummy0 198.18.0.1/24

1. Create the chain

Provision the root/netns peer pair and the dummy endpoint.

  1. sudo bash scripts/hum-dev-netns.sh up
  2. sudo bash scripts/hum-dev-netns.sh status

2. Confirm readiness

  • Look for peer recv-ready: yes.
  • Verify both SMAC64 trace IDs are populated.
  • Confirm root and netns addresses match the diagram.

3. Trace downstream state

Use trace mode when the plot suggests traffic or cadence changes.

  1. sudo bash scripts/hum-dev-netns.sh trace
  2. sudo bash scripts/hum-dev-netns.sh down
